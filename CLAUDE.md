**开发模式**: 渐进式开发

---

## 开发原则

### 1. 渐进式开发

**核心思想**: 每个阶段基于上一阶段的确认内容向后扩展

```
阶段1: demo.html 确认界面
   ↓
阶段2: 根据 demo 实现前端效果
   ↓
阶段3: 根据前端效果设计接口
   ↓
阶段4: 根据接口设计数据结构
   ↓
阶段5: 实现后端服务
```

**重要**: 不跳过阶段，不回溯修改已确认的内容

### 2. 内容确认原则

- **已确认的界面**: demo.html
- **已确认的接口**: docs/API接口设计文档.md
- **已确认的架构**: docs/架构与规范文档.md

**开发新功能时**:
1. 检查是否有上一阶段的确认内容
2. 基于确认内容进行扩展
3. 不破坏已有设计

### 3. Prompt 记录规范

**所有开发过程中的 prompt 必须记录在**: `docs/过程Prompt记录文档.md`

**记录格式**:
```markdown
## 节点 X：节点名称

**Prompt**: 具体的prompt内容
```

**何时记录**:
- 开始新阶段开发时
- 遇到需要决策的设计问题时
- 完成重要功能节点时

## 示例：渐进式开发流程

### 阶段1: 确认界面
- 完成 demo.html
- 所有交互以 demo.html 为准

### 阶段2: 实现前端
- 参考 demo.html
- 不改变交互逻辑
- 转换为小程序代码

### 阶段3: 设计接口
- 基于前端交互
- 设计 API 规范
- 不修改前端

### 阶段4: 实现后端
- 按照接口规范
- 不修改接口设计
- 实现业务逻辑

**关键**: 每个阶段不回溯修改上一阶段的内容

## 注意事项

### ⚠️ 禁止行为

1. **不要跳过阶段**
   - 不能在没有接口设计时直接写后端代码
   - 不能在没有界面确认时直接设计接口

2. **不要修改已确认内容**
   - 不要修改 demo.html 的交互
   - 不要修改已确定的接口规范
   - 不要破坏架构设计原则

3. **不要遗漏记录**
   - 不要忘记在过程Prompt文档中记录
   - 不要跳过重要节点的记录

### ✅ 推荐行为

1. **按阶段推进**
   - 每个阶段充分确认
   - 基于上一阶段扩展

2. **及时记录**
   - 记录所有重要决策
   - 记录设计过程

3. **保持一致**
   - 与已确认内容保持一致
   - 遵循设计原则
