# 过程 Prompt 记录文档

## 节点 1：确认 demo.html 界面和交互

**Prompt**：分析 demo.html 的完整用户交互流程，记录每个页面的所有用户操作和系统响应。

## 节点 2：根据 demo 实现前端效果

**Prompt**：将 demo.html 转换为符合 uni-app 小程序规范的前端代码，包括页面组件、样式、交互逻辑。

## 节点 3：根据前端效果完善前后端接口设计

**Prompt**：基于前端交互功能点，设计完整的前后端接口规范，确保覆盖所有用户操作场景。

## 节点 4：验证接口设计的完整性

**Prompt**：验证接口设计是否覆盖所有前端交互场景，检查是否有遗漏的功能点或数据流向问题。

## 节点 5：创建接口实现优先级

**Prompt**：根据前端交互的重要性和依赖关系，确定接口实现的优先级顺序，确保核心功能优先实现。

## 节点 6：实现后端服务框架

**Prompt**：搭建 Node.js + Express 基础项目，配置开发环境、中间件、日志系统。

## 节点 7：实现数据库系统

**Prompt**：选择数据库方案（sql.js/SQLite），创建数据模型（users、games、likes表），配置数据库连接。

## 节点 8：实现用户系统

**Prompt**：实现微信登录、JWT认证、用户信息管理接口。

## 节点 9：实现游戏管理接口

**Prompt**：实现游戏的创建、查询、更新、删除、发布、克隆等核心功能接口。

## 节点 10：实现 AI 生成服务

**Prompt**：集成 AI 大模型（OpenAI/Claude），实现 SSE 流式生成游戏代码接口。

## 节点 11：实现互动功能接口

**Prompt**：实现点赞、取消点赞、点赞状态查询、游玩记录等互动功能。

## 节点 12：前后端联调

**Prompt**：前端切换到真实 API，移除 Mock 数据，验证所有功能正常工作。

## 节点 13：测试与上线

**Prompt**：功能测试、性能测试、兼容性测试，准备小程序上线材料。
