# Demo.html 功能点分析

## 一、用户操作功能点

### 1. 我的作品页面
- checkBackend() - 检查后端服务状态
- switchTab('create') - 跳转到创作页面
- playGame(gameId) - 试玩游戏

### 2. 创作游戏页面
- toggleRecording() - 语音录制
- onDescriptionInput() - 输入描述
- generateGame() - AI 生成游戏（Stream 输出）
- testGame() - 试玩生成的游戏
- saveGame() - 保存游戏

### 3. 游戏广场页面
- playGame(gameId) - 试玩游戏
- switchTab() - 切换标签页

### 4. 游戏试玩页面
- closePlayPage() - 关闭试玩
- shareGame() - 分享游戏
- editGame(gameId) - 编辑游戏（我的游戏）
- publishGame(gameId) - 发布/设为私密（我的游戏）
- deleteGame(gameId) - 删除游戏（我的游戏）
- toggleLike(gameId) - 点赞/取消点赞（别人的游戏）
- cloneGame(gameId) - 克隆游戏（别人的游戏）
- followAuthor(authorId) - 关注作者

## 二、数据获取功能点

### 1. 静态数据
- 我的游戏列表（3个示例游戏）
- 游戏广场列表（3个公开游戏）
- 用户信息（模拟）
- 后端服务状态

### 2. 动态数据需求
- 游戏详情（根据 ID）
- 用户登录状态
- 点赞状态
- 配额信息（每日生成次数）

## 三、后端 API 需求清单

### 必需接口
1. GET /health - 健康检查
2. POST /api/user/login - 微信登录
3. GET /api/user/info - 获取用户信息
4. GET /api/games - 获取我的游戏列表
5. POST /api/games - 创建游戏
6. GET /api/games/:id - 获取游戏详情
7. PUT /api/games/:id - 更新游戏
8. DELETE /api/games/:id - 删除游戏
9. GET /api/games/public - 获取公开游戏列表
10. POST /api/games/:id/publish - 发布游戏
11. POST /api/games/:id/clone - 克隆游戏
12. POST /api/ai/generate - AI 生成游戏（SSE）
13. POST /api/ai/modify - AI 修改游戏（SSE）
14. POST /api/like - 点赞游戏
15. DELETE /api/like - 取消点赞
16. GET /api/like/:gameId - 检查点赞状态
17. GET /api/quota - 获取配额信息
