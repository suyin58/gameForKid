## 一、前端架构

### 1.1 分层架构

```
┌─────────────────────────────────────────┐
│           页面层 (Pages)                  │
│     index, create, plaza, play           │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│        组件层 (Components)                │
│   game-card, voice-input, webview...     │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│      状态管理层 (Store - Pinia)           │
│         user, game, app                  │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│       业务逻辑层 (Services & Utils)       │
│   api, request, voice, storage...        │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│        网络层 (Request)                   │
│      统一请求封装 + 拦截器                 │
└─────────────────────────────────────────┘
```

### 1.2 设计原则

**单一职责原则**
- 每个模块/文件只做一件事
- 页面负责展示和交互
- 组件负责UI复用
- Store负责状态管理
- Service负责业务逻辑

**依赖注入原则**
- 通过 Pinia 实现状态共享
- 页面依赖 Store，Store 依赖 Service
- 单向数据流

**分层隔离原则**
- 页面层不直接调用 API
- 通过 Store 和 Service 层隔离
- 便于测试和维护

### 1.3 技术栈

| 技术 | 版本 | 用途 |
|-----|------|-----|
| uni-app | 最新 | 跨平台框架 |
| Vue 3 | 3.x | 前端框架 |
| Pinia | 最新 | 状态管理 |
| SCSS | 最新 | 样式预处理器 |

## 二、后端架构

### 2.1 分层架构

```
┌─────────────────────────────────────────┐
│         路由层 (Routes)                   │
│    /api/user, /api/game, /api/ai...      │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│       控制器层 (Controllers)              │
│   处理请求参数，调用Service，返回响应     │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│        服务层 (Services)                  │
│   业务逻辑处理，数据聚合，外部API调用     │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│        数据访问层 (Models)                │
│   数据模型，数据库操作                   │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│           数据库 (Database)               │
│   users, games, likes, quota...          │
└─────────────────────────────────────────┘
```

### 2.2 设计原则

**分层架构原则**
- 每层只依赖下一层
- 路由层 → 控制器层 → 服务层 → 数据访问层
- 实现解耦和复用

**单一职责原则**
- Routes: 定义路由
- Controllers: 处理请求响应
- Services: 业务逻辑
- Models: 数据操作

**依赖倒置原则**
- 高层不依赖低层
- 通过接口抽象
- 便于测试和替换

| 技术 | 版本 | 用途 |
|-----|------|-----|
| Node.js | 18+ | 运行环境 |
| Express | 4.x | Web框架 |
| 数据库 | 待定 | 数据存储 |
| JWT | - | 认证 |
| Winston | - | 日志 |

## 九、接口规范

### 9.1 基础规范

**基础协议**
- 协议: HTTP / HTTPS
- 数据格式: JSON
- 字符编码: UTF-8
- 时间格式: ISO 8601 (YYYY-MM-DDTHH:mm:ss.sssZ)
- 时区: UTC

**基础URL**
```
开发环境: http://localhost:3001/api/v1
生产环境: https://api.kidsgame.com/api/v1
```

**API版本控制**
- 当前版本: v1
- 版本通过URL路径控制: `/api/v1/user/login`

### 9.2 请求规范

**必须包含的请求头**
```http
Content-Type: application/json
Accept: application/json
```

**认证请求头（需要登录的接口）**
```http
Authorization: Bearer <token>
```

**JWT Token 认证**
- 有效期: 开发环境30天，生产环境7天
- 过期后需重新登录
- 暂不支持刷新Token机制

**请求参数格式**
- URL参数: `/api/games/public?page=1&limit=20&type=casual`
- Request Body: JSON格式
- 分页参数: page（页码）、limit（每页数量，默认20，最大50）

### 9.3 响应规范

**统一响应格式**
```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

**响应状态码**
| code | 说明 |
|-----|------|
| 0 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未登录 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 429 | 配额不足 |
| 500 | 服务器错误 |
| 1001 | 内容违规 |
| 1002 | 游戏代码无效 |

**分页响应格式**
```json
{
  "code": 0,
  "data": {
    "total": 100,
    "page": 1,
    "limit": 20,
    "items": []
  }
}
```

### 9.4 错误处理规范

**错误响应格式**
```json
{
  "code": 400,
  "message": "错误描述",
  "data": null
}
```

**错误处理原则**
- 所有错误返回统一格式
- 前端根据 code 进行错误处理
- 敏感信息不暴露给前端
