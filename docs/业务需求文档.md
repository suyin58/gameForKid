# 儿童AI游戏创作平台 - 业务需求文档

## 文档信息

| 项目名称 | 儿童AI游戏创作平台 |
|---------|------------------|
| 项目类型 | 微信小程序 |
| 目标用户 | 不会编程的儿童（6-14岁） |
| 项目性质 | 个人实践项目 |
| 创建日期 | 2026-03-22 |
| 版本 | v1.0 |

---

## 一、项目概述

### 1.1 产品定位
一个面向儿童的游戏创作平台，通过**AI辅助**和**语音交互**，让不会编程的孩子也能发挥想象力，创作属于自己的HTML小游戏，并与他人分享和互动。

### 1.2 核心价值
- **降低创作门槛**：不会编程也能做游戏
- **激发想象力**：把想法变成现实
- **学习与模仿**：通过克隆他人作品学习创作
- **社交互动**：发布作品，获得认可

### 1.3 目标用户
- **主要用户**：6-14岁儿童
- **次要用户**：家长（监督和引导）

## 二、核心业务流程

### 2.1 创作与发布流程

```
┌─────────────────────────────────────────────────────────┐
│  用户A创作游戏                                           │
├─────────────────────────────────────────────────────────┤
│  1. 打开小程序 → 进入首页                                │
│  2. 点击「新创作」按钮                                    │
│  3. 按住说话按钮，语音描述游戏想法                        │
│     例："我想做一个跳跃游戏，主角是一只小兔子，            │
│         要躲避障碍物吃到胡萝卜"                           │
│  4. 语音转成文字，显示在输入框                            │
│  5. 查看并编辑文字（修正语音识别错误）                    │
│  6. 点击「生成游戏」                                      │
│  7. AI生成HTML游戏代码                                    │
│  8. 进入试玩页面，实际体验游戏                            │
│  9. 如果满意：                                            │
│     - 点击「保存」→ 保存到「我的作品」（私有）             │
│     - 点击「发布到广场」→ 所有人可见（公开）               │
│  10. 如果不满意：                                         │
│      - 修改描述 → 重新生成                                │
│      - 或调整游戏参数 → 重新生成                          │
└─────────────────────────────────────────────────────────┘
```

### 2.2 浏览与克隆流程

```
┌─────────────────────────────────────────────────────────┐
│  用户B浏览与克隆                                         │
├─────────────────────────────────────────────────────────┤
│  1. 打开小程序 → 进入首页                                │
│  2. 点击「游戏广场」                                      │
│  3. 浏览其他人发布的游戏                                  │
│  4. 看到感兴趣的游戏 → 点击试玩                           │
│  5. 体验游戏                                             │
│  6. 觉得很有趣，点击「克隆到我的作品」                     │
│  7. 游戏复制到「我的作品」列表                            │
│  8. 进入克隆的游戏，语音描述修改想法                      │
│     例："把兔子换成恐龙，胡萝卜换成肉"                     │
│  9. AI在原有游戏基础上修改                                │
│  10. 生成自己的版本，可以发布                             │
└─────────────────────────────────────────────────────────┘
```

## 三、功能需求

### 3.1 页面结构

```
首页（我的作品）
├── 新创作 → 创作页
├── 游戏广场 → 广场页
└── 游戏列表
    └── 点击游戏 → 试玩页

游戏广场
├── 游戏列表（所有人发布的）
└── 点击游戏 → 试玩页（带克隆按钮）

创作页
├── 语音输入 → 转文字
├── 文字编辑
└── 生成 → 试玩页

试玩页
├── 游戏画面（webview）
├── 保存（私有）
├── 发布到广场（公开）
├── 重新生成
└── 克隆到我的作品（广场游戏才有）
```

### 3.3 核心功能详细说明

#### 3.3.1 语音输入

**功能描述**：用户通过语音描述游戏想法，系统转成文字。

**技术实现**：
- 使用微信语音识别API（wx.startRecord, wx.translateVoice）
- 识别结果返回文字
- 支持中英文混合

**交互流程**：
1. 用户按住"按住说话"按钮
2. 开始录音，显示录音动画
3. 松开按钮，停止录音
4. 调用语音识别API
5. 识别结果显示在输入框
6. 用户可以编辑文字

**错误处理**：
- 识别失败：提示"没有听清，请再说一次"
- 网络错误：提示"网络连接失败"

---

#### 3.3.2 AI游戏生成

**功能描述**：根据用户描述，AI生成完整的HTML游戏代码。

**生成流程**：
1. 用户提交游戏描述
2. 后端接收描述
3. 调用大模型API（GPT-4/Claude/等）
4. 生成HTML游戏代码（包含HTML、CSS、JavaScript）
5. 保存游戏代码
6. 返回给前端展示

**生成内容要求**：
- 完整的单页HTML文件
- 界面色彩鲜艳，适合儿童
- 操作简单直观
- 包含游戏说明和玩法提示
- 响应式设计，适配手机屏幕

**游戏复杂度**：
- 简单：猜数字、点击计数、问答
- 中等：打地鼠、接东西、简单跳跃
- 复杂：多关卡、计分系统、音效
- 根据用户描述自动判断复杂度

**加载状态**：
```
+--------------------------------------+
|                                      |
|         🤖 AI正在为你创作游戏...        │
|                                      |
|         [加载动画 - 旋转彩虹圆圈]       │
|                                      |
|         预计等待时间：约 5 秒           │
|                                      |
+--------------------------------------+
```

**错误处理**：
- 生成失败：提示"生成失败，请重新描述"
- 生成超时：提示"生成时间过长，请稍后再试"

---

#### 3.3.3 游戏保存

**功能描述**：保存游戏到用户的"我的作品"列表。

**保存内容**：
- 游戏名称（默认AI生成，可修改）
- 游戏代码（HTML）
- 游戏描述（用户输入）
- 游戏类型（AI识别）
- 创建时间
- 可见性（私有/公开）

**数据库字段**：
```javascript
{
  id: String,           // 游戏ID
  userId: String,       // 用户ID
  title: String,        // 游戏名称
  description: String,  // 用户描述
  code: String,         // HTML游戏代码
  type: String,         // 游戏类型
  visibility: String,   // 可见性: 'private' | 'public'
  isCloned: Boolean,    // 是否克隆的
  clonedFrom: String,   // 克隆自哪个游戏ID
  createdAt: Date,      // 创建时间
  updatedAt: Date       // 更新时间
}
```

#### 3.3.5 游戏克隆

**功能描述**：将广场上的游戏复制到自己的作品列表，可以基于此继续创作。

**克隆流程**：
1. 用户在广场试玩游戏
2. 点击"克隆到我的作品"
3. 复制游戏代码到自己的作品
4. 标记克隆来源
5. 显示提示"已克隆到你的作品"

**克隆后的游戏**：
- 完整复制原游戏代码
- 保留原游戏名称（加后缀"的副本"）
- 记录原始作者信息
- 标记 `isCloned: true`
- 记录 `clonedFrom: 原游戏ID`

**基于克隆游戏继续创作**：
- 进入克隆的游戏
- 点击"继续创作"
- 跳转到创作页，预填充原描述
- 用户语音添加新需求
- AI在原游戏基础上修改

**示例**：
```
原游戏: "我想做一个跳跃游戏，主角是一只小兔子，要躲避障碍物吃到胡萝卜"

克隆后语音: "把兔子换成恐龙，胡萝卜换成肉，速度加快一点"

AI生成: 在原游戏代码基础上修改角色、道具和速度参数
```

**克隆溯源显示**：
```
游戏详情页：
┌──────┐
│ 🦖   │ 跳跃吃肉（修改版）
│ 跳跃  │ 基于: 小明的《跳跃吃胡萝卜》
│ 吃肉  │ 创建于 1天前
└──────┘
```

### 3.4 用户系统

#### 3.4.1 微信登录

**功能描述**：用户通过微信授权登录。

**登录流程**：
1. 用户打开小程序
2. 检查登录状态
3. 未登录 → 调用微信登录（wx.login）
4. 获取用户信息（昵称、头像）
5. 后端创建/更新用户
6. 返回用户ID
7. 保存登录状态

**用户信息存储**：
```javascript
{
  _id: String,         // 用户ID
  openid: String,      // 微信openid
  nickname: String,    // 昵称
  avatar: String,      // 头像URL
  createdAt: Date,     // 注册时间
  lastLoginAt: Date    // 最后登录时间
}
```

---

#### 3.4.2 用户身份

**显示规则**：
- 首页显示用户昵称
- 游戏广场显示作者昵称
- 游戏详情显示作者信息

**隐私保护**：
- 不显示用户真实姓名
- 不显示手机号等敏感信息
- 用户可设置是否在广场显示

### B. Prompt模板示例

**基础模板**：
```
你是一个专业的儿童游戏开发者。请根据以下描述生成一个完整的HTML小游戏。

用户描述：{用户描述}

要求：
1. 生成单页HTML文件，包含HTML、CSS、JavaScript
2. 界面色彩鲜艳，适合6-14岁儿童
3. 操作简单直观，支持手机触摸操作
4. 包含游戏说明和玩法提示
5. 响应式设计，适配手机屏幕

请直接输出HTML代码，不要其他说明文字。
```

**修改模板**（基于克隆游戏）：
```
你是一个专业的儿童游戏开发者。用户想在现有游戏基础上做修改。

原游戏描述：{原游戏描述}
原游戏代码：{原游戏代码}
修改需求：{修改需求}

要求：
1. 在原游戏代码基础上修改
2. 保持原有游戏框架
3. 只修改用户要求的部分
4. 确保修改后的游戏可以正常运行

请直接输出修改后的完整HTML代码。
```
