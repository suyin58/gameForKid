# 配额系统使用指南

## 📋 概述

本系统实现了基于数据库的用户每日配额管理，用于限制AI生成和修改的使用次数。

## ✅ 集成状态

配额系统已完全集成到主应用（app.js）中，包括：
- ✅ 数据库模型（quota.model.js）
- ✅ 业务逻辑服务（quota.service.js）
- ✅ 配额检查中间件（quota.middleware.js）
- ✅ 配额查询路由（quota.routes.js）
- ✅ AI生成/修改路由（ai.routes.js）

## 📊 数据库表结构

### user_daily_quota 表

```sql
CREATE TABLE user_daily_quota (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  date TEXT NOT NULL,
  generate_count INTEGER DEFAULT 0,
  modify_count INTEGER DEFAULT 0,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  UNIQUE(user_id, date)
);
```

**字段说明：**
- `user_id`: 用户唯一标识
- `date`: 日期（格式：YYYY-MM-DD）
- `generate_count`: 当日AI生成次数
- `modify_count`: 当日AI修改次数
- `created_at`: 记录创建时间
- `updated_at`: 记录更新时间

**索引：**
- `UNIQUE(user_id, date)`: 确保每个用户每天只有一条记录
- `INDEX(date)`: 用于批量清理过期数据

### 2. 创建配额中间件

```javascript
const createQuotaMiddleware = require('./src/middlewares/quota.middleware');

// 在app.js中
const db = require('./src/models/database');
const quotaMiddleware = createQuotaMiddleware(db);

// AI生成接口 - 检查并扣除生成配额
app.post('/api/v1/ai/generate',
  authMiddleware.auth,
  quotaMiddleware.checkGenerateQuota,  // 请求前检查配额
  aiController.generateGame,
  quotaMiddleware.deductGenerateQuota   // 响应后扣除配额

// AI修改接口 - 检查并扣除修改配额
app.post('/api/v1/ai/modify',
  authMiddleware.auth,
  quotaMiddleware.checkModifyQuota,
  aiController.modifyGame,
  quotaMiddleware.deductModifyQuota
);
```

## ⚠️ 配额不足错误处理

当用户配额用完时，返回以下错误：

```json
{
  "code": 5004,
  "message": "今日AI生成配额已用完（10/10）",
  "data": {
    "quotaInfo": {
      "generate": {
        "used": 10,
        "remaining": 0,
        "limit": 10
      },
      "resetAt": "2026-03-24T00:00:00.000Z"
    }
  }
}
```

**前端处理建议：**

```javascript
try {
  await api.ai.generateGame(description);
} catch (error) {
  if (error.code === 5004) {
    // 配额不足
    const remaining = error.data.quotaInfo.generate.remaining;
    const resetAt = error.data.quotaInfo.resetAt;

    uni.showModal({
      title: '配额已用完',
      content: `今日生成次数已用完，剩余次数将在明天凌晨${new Date(resetAt).getHours()}:00重置。升级VIP可获得更多配额哦！`,
      confirmText: '升级VIP',
      cancelText: '我知道了',
      success: (res) => {
        if (res.confirm) {
          // 跳转到VIP升级页面
          uni.navigateTo({ url: '/pages/vip' });
        }
      }
    });
  }
}
```

## 🧪 测试配额系统

### 单元测试示例

```javascript
const QuotaService = require('../src/services/quota.service');
const db = require('../src/models/database');

describe('配额服务测试', () => {
  let quotaService;
  let testUserId = 999;

  beforeAll(async () => {
    const database = await db.init();
    quotaService = new QuotaService(database);
    await quotaService.init();
  });

  test('检查免费用户生成配额', async () => {
    const hasQuota = await quotaService.checkGenerateQuota(testUserId, 'free');
    expect(hasQuota).toBe(true);
  });

  test('使用生成配额', async () => {
    const result = await quotaService.useGenerateQuota(testUserId, 'free');
    expect(result.used).toBeGreaterThan(0);
    expect(result.remaining).toBeLessThan(10);
  });

  test('配额用完抛出错误', async () => {
    // 快速用完10次配额
    for (let i = 0; i < 10; i++) {
      await quotaService.useGenerateQuota(testUserId, 'free');
    }

    await expect(
      quotaService.useGenerateQuota(testUserId, 'free')
    ).rejects.toThrow(/今日AI生成配额已用完/);
  });
});
```

## 🚀 性能优化

### 使用缓存减少数据库查询

```javascript
const NodeCache = require('node-cache');
const quotaCache = new NodeCache({ stdTTL: 60 }); // 缓存1分钟

async getUserQuotaInfo(userId, quotaType = 'free') {
  const cacheKey = `quota:${userId}:${new Date().toISOString().split('T')[0]}`;

  // 先查缓存
  const cached = quotaCache.get(cacheKey);
  if (cached) {
    return cached;
  }

  // 查询数据库
  const quotaInfo = await this.quotaModel.getUserQuota(userId, today);
  const config = this.getQuotaConfig(quotaType);
```
